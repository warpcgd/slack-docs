# 开发者指南

# 本地开发

slack通过 lerna 管理各个包

按照下面的步骤即可本地开发slack

# 克隆仓库
# 默认为 dev 分支，包含 slack 的当前版本代码

git clone https://git.duowan.com/webs/ikaixindou/common/slack.git

# 切换分支到 dev

git checkout dev

# 安装依赖
yarn && yarn bootstrap

# 构建依赖
yarn dev

# 构建demo页面，浏览器访问 http://dev.ihago.net:8888/#/
yarn dev:demo

# 构建docs页面，浏览器访问 http://localhost:8080/a/slack-docs/guide/
yarn dev:docs

# 目录结构

  • 仓库的组件代码位于 packages 下，每个组件一个文件夹
  • docs 目录下是文档网站的代码，本地开发时可以在目录下运行 npm run dev:docs 开启文档网站
slack
├─ build     # Rollup构建
├─ packages  # 子包
   ├─ docs      # 文档
   ├─ business  # 业务组件
   ├─ ui        # 通用组件
      ├─ build      # 构建
      ├─ src        # 组件
      ├─ test       # 测试
      ├─ types      # 类型
   ├─ use       # 通用工具
├─ plugins   # Rollup插件
└─ types     # 类型

# 提交

# 分支

  • master 主 release 分支，测试 + 构建 + npm 发布 + 文档构建
  • hotfix master 修复分支，修复 master 线上问题
  • alpha 主 alpha 测试分支，测试 + 构建 + npm 发布 + 文档构建
  • dev 主开发分支， 目前开发版本
  • feature-* 各个功能分支， 目前开发的功能
  • next 下一版本分支，用于 next 测试
  • v* 主版本分支，各版本稳定版本

# 提交流程

  • dev 主开发分支，可以合并至 alpha 和 master，每次合并前需 merge master 保证版本号最新
  • feature 功能分支，可以合并至的 dev，每次合并前需 merge dev 保证代码更新
  • alpha 测试分支，主要为测试大版本更新及重要功能，只能合并开发分支，再合并到 master，单向流程
  • master 主发版分支，稳定版本分支，只能通过 Gitlab merge request合并，code view 后再进行发版
  • hotfix 主版本修复分支，修复 master 问题后，合并前需 merge master 保证版本号最新
